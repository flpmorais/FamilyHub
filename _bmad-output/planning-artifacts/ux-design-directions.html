FamilyHub — Design Directions

6 visual directions for the Vacation Packing List — core V1 screen. Same content, different approaches.

Direction 1 — Material 3 Classic

Recommended baseline

Standard M3 implementation. Status strips as 4dp left borders on flat cards. Filter chips in a horizontal scroll row. Context-sensitive bottom bar with FAB. Clean, familiar Android patterns — the full M3 spec, applied precisely to FamilyHub's content model.

Direction 2 — Card Groups by Category

Category-first hierarchy

Items grouped into elevated cards by category. Status shown as coloured dots. At-a-glance count bar across the top shows distribution by status. Heavier visual weight per card; better for scanning what remains per category. Less compact — fewer items visible without scrolling.

Direction 3 — Dark Mode First

Immersive / OLED-friendly

Dark M3 surface colour scheme — deep warm charcoal base (#1A1110) with the terracotta and peach accents glowing. Status colours stay vivid against dark. Uses a donut ring progress indicator in the header. Appeals to night-use patterns (planning a trip in the evening). Pairs with a light-mode option as theme toggle.

Direction 4 — Status Tabs (Top Navigation)

Status as primary nav

Status becomes the primary navigation dimension — each status gets a tab with its count. Swiping between tabs switches the active status filter. Inside each tab, category subgroups. Good for power users who mentally model by status. Risk: 6 tabs is wide; relies on horizontal swipe gesture which competes with item swipe actions.

Direction 5 — Minimalist / Content First

Ultra-clean, text-forward

Maximum content density. No card elevation — flat items with generous line spacing. Status as coloured dot only (no badge, no strip). Progress shown as a segmented colour bar. Breathes more than D1 — better for reading, less clear at a glance than strip+badge. Works when you trust colour memory for status.

Direction 6 — Bold Headers + Status Swimlanes

High visual contrast

Full-width terracotta header with bold trip title and aggregated stats (problems / to-buy / remaining). Items in rounded swimlane cards grouped by status. Each lane has a coloured dot + count. High contrast, very scannable — immediately shows what needs attention. Slightly heavier than D1 but visually distinctive.

All Directions — Side-by-Side Comparison

Quick reference for key trade-offs across all six directions.

Direction Theme Status visibility Compact density Filter power Category grouping Risk
D1 — M3 Classic Light ● Strip + badge — excellent ✓ 56dp rows Right panel — full combinatorial Subheadings within flat list None — proven M3 baseline
D2 — Card Groups Light ● Dot only — moderate ✗ Cards = 80–100dp per item Stacked bar + chips Cards as primary grouping Fewer items on screen; less scannable status
D3 — Dark Mode Dark ● Strip + text label — excellent ✓ 56dp rows Right panel Subheadings Not all users prefer dark; needs theme toggle
D4 — Status Tabs Light ● Tab = filter by status directly ✓ 56dp rows Tabs only — no cross-dimension filter Subheadings within tab Tab swipe conflicts with item swipe; 6 tabs = crowded
D5 — Minimalist Light ● Dot only — requires colour memory ✓ Flattest — most items visible Needs filter panel addition Section headers Low discoverability of status without labels
D6 — Bold Swimlanes Light + bold header ● Swimlane = status bucket — excellent ✗ Cards + spacing = less dense Swimlanes only — no cross-filter Status is group, not category Category grouping requires secondary filter mode
Design Evaluation Guide
Layout Intuitiveness
Which information hierarchy feels most natural when you're at the supermarket buying missing items? When you're packing the night before departure?
Visual Weight
With 50+ items, which density feels right? Too heavy = exhausting to scan. Too light = status colours aren't visible enough.
Brand Alignment
Does this look and feel like something the family would be proud to use? Warm, personal, family-forward — not a corporate tool.
Angela's Test
Angela has no builder bias. Which direction would she open, look at, and immediately understand without explanation?

★ Chosen Direction — FamilyHub Combined

D1 cards · D2 header counters · FAB only · Teal last-minute

Flat M3 cards with 4dp status strip — no category/person grouping, pure flat list. D2-style header with trip name + status count pills that act as quick filters (tap to toggle). Floating FAB replaces the bottom bar entirely. Right-side filter panel for full combinatorial filtering. Last-Minute recoloured to teal (#00897B) — calm/handled, not urgent.

Default — flat list, no active filter
